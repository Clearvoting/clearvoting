# Member Voting Profile — Design Document

**Date:** 2026-03-05
**Status:** Approved

## Problem

When users search for their congressman on ClearVote, they see basic info (name, photo, service history, sponsored legislation count) but nothing about how that person actually votes. Voting record is the most useful information for understanding what a representative does in office.

## Solution

Add a full voting profile to the member detail page with three sections: voting statistics, recent key votes with AI-generated plain-language descriptions, and issue-based filtering.

## Core Principles

- **No bias:** No party-line comparisons, no "voted with party X%" stats. Party toggle remains hidden by default.
- **Plain language:** AI-generated one-liner descriptions for each bill voted on (cached after first generation).
- **Facts first:** Show the vote, the result, the date. Let users form their own opinions.

## Design

### Section 1: Voting Statistics

Displayed below the member header, above the vote list.

- **Participation rate** — % of votes they showed up for (donut chart, reuse existing component)
- **Total votes cast** this Congress
- **Yea / Nay ratio** — simple donut chart
- **Missed votes** count

No party comparisons. No scoring. Just raw participation and voting pattern data.

### Section 2: Recent Key Votes

Scrollable list of the member's last ~20 votes. Each entry shows:

| Field | Description |
|-------|-------------|
| Bill number | e.g., "H.R. 1" |
| One-liner | AI-generated plain-language description |
| Vote | Yea / Nay / Not Voting (color-coded, existing palette) |
| Date | When the vote occurred |
| Result | Passed / Failed |

Clicking a bill navigates to the existing bill detail page.

Vote colors (existing palette):
- Yea: #5B8A72 (green)
- Nay: #8A5B5B (red)
- Not Voting: #6B6B6B (gray)

### Section 3: Issue Filter

Horizontal row of tag/chip buttons for issue categories:
- Healthcare, Defense, Immigration, Taxes, Education, Environment, Economy, etc.
- Uses Congress.gov policy area classifications (already available in bill data)
- Clicking a category filters the vote list to that topic
- "All" chip selected by default

### Demo Mode

Mock voting records for the 4 FL demo members (Rick Scott, Ashley Moody, Byron Donalds, Michael Waltz) using real vote data from the 119th Congress. Each member gets 15-20 mock votes with realistic one-liners and issue categories.

### Real Mode

- **Senate votes:** Already parsed from Senate.gov XML. Cross-reference member bioguide IDs against vote records.
- **House votes:** Congress.gov API provides House vote data. Already have endpoint stub.
- **One-liners:** Generated by Claude (same no-adjective prompting), cached per bill.
- **Issue categories:** From Congress.gov bill policy areas and legislative subjects.

## What Stays the Same

- Party toggle behavior (hidden by default, click to reveal)
- Existing bill detail pages (voting profile links into them)
- Dark theme, midnight blue + gold palette, Inter/Playfair Display fonts
- No scoring, no ratings, no party comparisons
- Flat-file JSON caching

## Data Flow

```
Member page loads → fetch /api/members/detail/{id}
                  → fetch /api/members/{id}/votes (NEW)
                  → render stats + vote list
                  → user clicks issue chip → filter client-side
                  → user clicks bill → navigate to /bill?congress=&type=&number=
```

## New API Endpoint

```
GET /api/members/{bioguide_id}/votes?congress=119&limit=20&offset=0
```
